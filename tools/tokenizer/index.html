What is a token?

Language models don't read characters or words — they read tokens: chunks of bytes learned by a compression algorithm (Byte-Pair Encoding). A token is often a whole common word, a word with its leading space ( the), a subword fragment, a punctuation mark, or even a single byte of a multi-byte character like an emoji.

Each token maps to one integer ID. Model context windows, API pricing, and rate limits are all counted in tokens — not words — so the same sentence can cost differently across encodings. As a rough English rule of thumb, 1 token ≈ 4 characters ≈ 0.75 words, but code, other languages, and emoji break that ratio.

The colored chunks above are the exact tokenizer output: the text is encoded to IDs, then each ID is decoded back individually to recover its precise substring — including leading spaces. A dimmed chunk (□) is a token holding only partial bytes of a multi-byte character; the character shows up on the token that completes it.
