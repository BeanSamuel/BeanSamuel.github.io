How this works & what to trust

Token counts. OpenAI models (GPT-4o, GPT-4.1, o-series, GPT-4, GPT-3.5) are tokenized with the exact Byte-Pair Encoding they use in production (o200k_base / cl100k_base via gpt-tokenizer), so those counts are precise. Claude and other non-OpenAI models don't ship a browser tokenizer, so they're approximated with o200k_base and flagged ≈. For exact Claude counts, use Anthropic's count_tokens API.

Cost. Estimated total = (input tokens ÷ 1M × input price) + (expected output tokens ÷ 1M × output price). Set the two prices per model and the expected output length; the table recomputes live. Nothing is sent anywhere — no API key, no network calls beyond loading the tokenizer library once.
